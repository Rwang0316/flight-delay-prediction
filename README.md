# Flight Delay Prediction

**DSC 148: Introduction to Data Mining — Course Project**
Author: Richard Wang (Riw009@ucsd.edu)

Predicts whether a U.S. domestic flight will arrive **more than 15 minutes late**
(the DOT standard for a delayed flight), using **only information known before the
flight departs**. The project frames this as a binary classification problem,
compares Logistic Regression and Random Forest baselines against a LightGBM final
model, and addresses the severe class imbalance in the data.

## Getting the Data

This project uses the **2015 Flight Delays and Cancellations** dataset from the
U.S. Department of Transportation (Bureau of Transportation Statistics),
distributed on Kaggle.

The dataset comes as three files:

| File | Size | In this repo? |
|------|------|---------------|
| `flights.csv` | ~580 MB | **No** — too large for GitHub (100 MB limit) |
| `airlines.csv` | ~1 KB | Yes |
| `airports.csv` | ~25 KB | Yes |

### Why `flights.csv` is not in the repo

GitHub rejects any single file larger than 100 MB. The main `flights.csv` file is
roughly 580 MB, so it is intentionally excluded via `.gitignore`. The two small
lookup tables are included so you only need to download the one large file.

### How to download `flights.csv`

**Option A — Manual download (simplest):**

1. Go to https://www.kaggle.com/datasets/usdot/flight-delays
2. Sign in with a free Kaggle account and click **Download** (downloads a `.zip`).
3. Unzip it and copy **`flights.csv`** into the `data/` folder of this project.
   (You can ignore `airlines.csv` and `airports.csv` from the zip — they are
   already in the repo, though overwriting them is harmless.)

**Option B — Kaggle API (command line):**

1. Install the Kaggle CLI: `pip install kaggle`
2. Create an API token: Kaggle → Account → **Create New API Token**. This
   downloads `kaggle.json`. Place it at `~/.kaggle/kaggle.json` (Mac/Linux) or
   `C:\Users\<you>\.kaggle\kaggle.json` (Windows).
3. From the project root, run:
   ```bash
   kaggle datasets download -d usdot/flight-delays -p data --unzip
   ```
   This downloads and unzips all three CSVs directly into `data/`.

After either option, your `data/` folder should contain all three CSVs and the
notebook will run without modification.

## Running the Project

Open `flight_delay_prediction.ipynb` in VS Code (or Jupyter), select your Python
kernel, and **Run All**.

The notebook will:
1. Load and merge the three CSVs
2. Perform exploratory data analysis (saved to `figures/`)
3. Construct the target and remove data-leakage columns
4. Engineer temporal and historical airport delay-rate features
5. Train and evaluate Logistic Regression, Random Forest, and LightGBM
6. Run an ablation study, hyperparameter sensitivity sweep, and feature-importance analysis

All plots are written to `figures/` and all metric tables to `results/`.

> **Note:** The full dataset has ~5.7M rows. The notebook samples 600,000 rows
> (fixed seed) for fair, reproducible model comparison. Lower `SAMPLE_SIZE` in the
> sampling cell if your machine is memory-constrained.

## Method Summary

- **Target:** `is_delayed = ARRIVAL_DELAY > 15` (DOT standard)
- **Leakage prevention:** all post-departure columns (actual times, taxi times,
  delay-cause attributions) are excluded
- **Features:** month, day of week, scheduled departure time/duration, distance,
  airline, origin/destination airports, plus engineered hour-of-day, time-of-day,
  season, weekend flag, and origin/destination historical delay rates
- **Class imbalance:** 82/18 split handled with balanced class weighting
- **Models:** Logistic Regression and Random Forest baselines; LightGBM final model
- **Primary metric:** F1 on the delayed class (accuracy alone is misleading here)
